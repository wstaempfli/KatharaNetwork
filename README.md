# Computer Networks 2025: Project Kathara

This project aims to provide you with hands-on experience in network engineering through a series of interactive labs and exercises. The project not only covers the theoretical foundations of network protocols like OSPF, BGP and SCION, but also guides you through network configurations and troubleshooting scenarios you might encounter in real deployments. By engaging with these labs, you’ll build a solid understanding of how modern networks are designed, implemented, and maintained, preparing you for challenges you might eventually face in industry.

This is the first year that we have this project - we hope you enjoy exploring the covered concepts, protocols and technologies as much as we enjoyed putting this material together.

**On the Use of Generative AI:**
Generative AI has become a valuable tool in computer science and beyond. Services like Github Copilot and ChatGPT can speed up coding tasks significantly, though they pose the risk of its user not understanding their output, incorrect and hard to debug code, and generally reduced code quality. When used ethically however, for example by asking conceptual questions, these tools can strengthen your understanding of complex networking concepts and help you deepen your knowledge. We endorse this ethical use of such tools, while strongly rejecting the lazy use of generative AI, for example the generation of entire blocks of code/configuration.

Through extensive testing, we can say with confidence that generative AI tools struggle with network configuration on Linux and FRR. If you blindly trust their outputs, you will likely be lead down the wrong track more than once - thus wasting your time and effort. Instead, we suggest you only use these tools to gain a better understanding, and only use the extensive official documentation to solve the tasks.

*In summary:* We allow the use of generative AI for the purpose of asking conceptual/knowledge questions. Examples include "Explain why I assign IP addresses to interfaces rather than hosts?", "What is a collision domain?" and "What are costs in OSPF?". However, we forbid the use of generative AI to generate entire configuration blocks for you, and to answer questions we ask in the labs.

# Project Structure

This project is split into two phases, with both subdivided into four labs, each focusing on a different aspect of network engineering. The labs increase in complexity, so we recommend completing them in sequence.

Each lab is contained in its own dedicated folder and includes a TASK.md file. These task files contain both hands-on implementation/configuration assignments as well as questions that test your understanding. Please write your answers to the questions in the fields provided. Make sure to not perform any other modifications to the file.

In the [DOCS.md](./DOCS.md) file, we provide you with extensive documentation about the technologies you use throughout the labs. We encourage you to thoroughly read this document before starting, and to reference it frequently as you work through the labs. This resource will likely prove invaluable.
